# W1M3 - GDP ETL 파이프라인

## 학습 목표
- 웹사이트에서 데이터를 가져와서 요구사항에 맞게 가공하는 ETL 파이프라인을 만든다.
- Web Scraping에 대한 이해
- Pandas DataFrame에 대한 이해
- ETL Process에 대한 이해
- Database & SQL 기초

## 사전지식 (시나리오)
당신은 해외로 사업을 확장하고자 하는 기업에서 Data Engineer로 일하고 있다. 경영진에서 GDP가 높은 국가들을 대상으로 사업성을 평가하려고 한다.

이 자료는 앞으로 경영진에서 지속적으로 요구할 것으로 생각되기 때문에 자동화된 스크립트를 만들어야 한다.

## 기능 요구사항
- IMF에서 제공하는 국가별 GDP를 구한다. (https://en.wikipedia.org/wiki/List_of_countries_by_GDP_%28nominal%29)
- 국가별 GDP를 확인할 수 있는 테이블을 만든다.
  - GDP가 높은 국가들이 먼저 나와야 한다.
  - GDP의 단위는 1B USD, 소수점 2자리까지 표시한다.
  - IMF에서 매년 2회 자료를 제공하므로, 정보가 갱신되어도 코드를 재사용해서 정보를 얻을 수 있어야 한다.

### 화면 출력
- GDP가 100B USD 이상이 되는 국가만을 구해서 화면에 출력한다.
- 각 Region별로 top5 국가의 GDP 평균을 구해서 화면에 출력한다.

## 프로그래밍 요구사항

### 라이브러리 사용
- Web scraping은 BeautifulSoup4 라이브러리를 사용한다.
- 데이터 처리를 위해 pandas 라이브러리를 사용한다.
- 로그 기록 시 datetime 라이브러리를 사용한다.

### 코드 가독성
- 주석을 사용해서 설명을 추가한다.
- 함수를 만들어서 가독성과 재사용성을 높인다.

### 파일 포맷, 데이터베이스 이름 등
- 처리한 정보는 `Countries_by_GDP.json` 이름의 JSON 파일 포맷으로 저장한다.
- 필요한 모든 작업을 수행하는 `etl_project_gdp.py` 코드를 작성한다.

### ETL 프로세스
- ETL 프로세스에 따라 코드를 작성하고 각 단계의 시작과 끝을 로그에 기록한다.
- 모든 처리 과정은 `etl_project_log.txt` 로그 파일에 기록되어야 한다. (매번 새로 생성하지 않고 기존 파일에 append)
- 로그는 `time, log` 형식으로 기록한다. 시간은 `Year-Monthname-Day-Hour-Minute-Second` 포맷.

## 팀 활동 요구사항
- Wikipedia 페이지가 아닌, IMF 홈페이지에서 직접 데이터를 가져오는 방법은 없을지 토의한다.
- 데이터가 갱신되면 과거의 데이터는 어떻게 되어야 하는지, 과거 데이터 조회가 필요하다면 ETL 프로세스를 어떻게 변경해야 하는지 토의한다.
- 해당 내용을 팀위키에 기록하고 제출한다.

## 추가 요구사항
코드를 수정해서 아래 요구사항을 구현한다.
- 처리한 데이터를 데이터베이스에 저장한다. `Countries_by_GDP` 테이블명으로 `World_Economies.db` 데이터베이스에 저장하며, 해당 테이블은 `Country`, `GDP_USD_billion` 어트리뷰트를 반드시 가져야 한다.
- 데이터베이스는 sqlite3 라이브러리를 사용해서 만든다.
- 필요한 모든 작업을 수행하는 `etl_project_gdp_with_sql.py` 코드를 작성한다. (기존 코드와 별도로 2개의 코드를 제출)

### 화면 출력
- SQL Query를 사용해야 한다.
- GDP가 100B USD 이상이 되는 국가만을 구해서 화면에 출력한다.
- 각 Region별로 top5 국가의 GDP 평균을 구해서 화면에 출력한다.
