# W2M3 - multiprocessing.Queue

> 파이썬에 있는 `multiprocessing.Queue`가 있는 이유는 프로세스 간의 변수들은 서로 공유를 못하는데 이를 가능하게 하기 위해서 생겨났다는 것을 알게됐다.

## 학습 목표
- FIFO(선입선출) 방식으로 아이템을 올바르게 push/pop하는 스크립트를 작성하여, 파이썬의 `multiprocessing.Queue`에 대한 이해도를 보여준다.

## 기능 요구사항
- Push 연산:
  - 큐에 아이템을 추가하기 시작할 때 이를 알려야 한다.
  - 주어진 리스트의 각 색상을 큐에 추가해야 한다.
  - 아이템이 큐에 추가될 때, 아이템 번호와 아이템 자체를 표시해야 한다.
- Pop 연산:
  - 큐에서 아이템을 제거하기 시작할 때 이를 알려야 한다.
  - 큐가 빌 때까지 아이템을 하나씩 제거해야 한다.
  - 아이템이 큐에서 제거될 때, 아이템 번호와 아이템 자체를 표시해야 한다.

## 프로그래밍 요구사항
- 파이썬의 `multiprocessing` 모듈을 사용한다.
- 큐에 push할 아이템 목록을 정의한다.
- `Queue` 클래스를 사용하여 큐를 생성한다.
- 목록의 아이템들을 큐에 push한다.
- 큐가 빌 때까지 아이템을 pop한다.
- push와 pop 연산마다 아이템 번호와 아이템을 출력한다.

## 예상결과 및 동작예시
```
pushing items to queue:
item no: 1 red
item no: 2 green
item no: 3 blue
item no: 4 black

popping items from queue:
item no: 0 red
item no: 1 green
item no: 2 blue
item no: 3 black
```
