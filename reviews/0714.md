## 배운 것
- `docker exec -it`로 스크립트를 실행하면, 스크립트가 끝나며 터미널 세션이 종료될 때 SIGHUP 신호가 자식 프로세스(RM)에 전달됨. Java가 이를 종료 신호로 받아들여 RM이 뜨자마자 죽는 문제가 발생함
  → `start-dfs.sh`/`start-yarn.sh`를 `setsid`로 감싸서 터미널 세션과 분리하면 해결됨
- `yarn-site.xml`에 존재하지 않는 옛 호스트명(`hadoop-master`)이 남아있으면 RM이 웹 UI 포트를 열지 못하고 죽음
  → `yarn.resourcemanager.hostname`/`webapp.address`를 실제 호스트명(`namenode`)으로 맞춰야 함
  → 이 설정을 이전에 고쳐뒀었는데, 다른 부분을 수정하다가 실수로 같이 지워졌던 것이 원인이었음. 설정 수정 시 관련 없는 부분까지 함께 지워지지 않았는지 확인이 필요함
